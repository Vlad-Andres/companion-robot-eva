## Local API Contracts

This repo is designed to call locally hosted APIs (same machine or LAN) for:

- Vision recognition
- Speech-to-text
- Decision / action planning (LLM)

Configuration lives in [config.py](file:///workspace/config.py#L46-L71).

### Vision API

Config: `VisionAPIConfig` (`base_url`, `endpoint`, `timeout_seconds`)

Caller: [VisionClient._call_api](file:///workspace/perception/vision_client.py#L121-L152)

- Request:
  - `POST <base_url><endpoint>`
  - Body: bytes (recommended: JPEG or PNG)
  - `Content-Type: application/octet-stream`
- Response JSON:
  - `{"objects": [{"label": "...", "position": {"x": 0, "y": 0, "width": 0, "height": 0}, "confidence": 0.0}]}`

### Speech-to-Text (STT) API

Config: `SpeechAPIConfig` (`base_url`, `endpoint`, `timeout_seconds`)

Caller: [SpeechClient._call_api](file:///workspace/perception/speech_client.py#L126-L167)

- Request:
  - `POST <base_url><endpoint>`
  - Body: audio bytes (current assumption: PCM16LE, mono, 16 kHz)
  - `Content-Type: audio/pcm`
- Response JSON:
  - `{"text": "..."}` (empty string allowed)

Practical recommendation: if you’re building the STT API from scratch, start with PCM16LE for simplicity and speed. If you want plug-and-play tools, accept WAV (`audio/wav`) instead.

### Decision (LLM) API

Config: `DecisionAPIConfig` (`base_url`, `endpoint`, `timeout_seconds`, `max_history_turns`)

Caller: [DecisionEngine._call_llm_api](file:///workspace/decision/decision_engine.py#L183-L197)

- Request:
  - `POST <base_url><endpoint>`
  - JSON body generated by [ContextBuilder](file:///workspace/decision/context_builder.py#L45-L93)
- Response JSON:
  - `{"actions": [ ... ]}`

Each action item must match the schema in [action_types.py](file:///workspace/actions/action_types.py#L32-L117):

- `{"type": "speak", "payload": {"text": "Hello"}}`
- `{"type": "set_eye_expression", "payload": {"expression": "happy"}}`
- `{"type": "play_eye_animation", "payload": {"animation": "BLINK_SHORT"}}`
